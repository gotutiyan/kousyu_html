サイト制作をするにあたって

htmlを軸にサイト制作をするにあたっての環境構築的なことを書きます。

  • htmlのコードは、.htmlの拡張子のファイルに記述していきます。つまり、メモ帳か何かでテキストファイルを作り、.txtのファイル名を.htmlに書き換えるだけで準備は完了です。
    （text.txt　→ text.htmlみたいな）

  • htmlの実行（というか、書いたものがどう表示されるかを確認する）時には、.htmlのファイルをブラウザで開くだけで行えます。この操作は、デフォルトでは.htmlファイルをダブルクリックするだけで行えると思います。C言語みたいなコンパイルとかいう作業は必要ありません。

  • ファイルの編集はメモ帳でもターミナル上のvimでも、文字を書くことができればなんでもokです。しかし、できるだけVSCodeやAtomなどのIDEを使うことをお勧めします(コード補完が楽なので。)

  • これ見落としがちなんですが、そのサイトにおけるいわゆるトップページに当たるファイル名は、index.htmlとします。作ったサイトを本当にサーバーに置いて全世界から閲覧できるようにするとき、数あるhtmlファイルの中からサーバーが一番最初に探すのはindex.htmlだからですね。

  • htmlはプログラミング言語ではないです。（入力を受けて云々みたいな処理が無いので、ただのマークアップ言語です。）

基本的なテンプレ

基本的なテンプレ、つまり　htmlの最小構成要素は以下の通りです。

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title></title>
    </head>
    <body>
    </body>
</html>

html,head,bodyといったものは全てタグと呼ばれるもので、後述します。charsetを使って文字コードをutf-8にすることで、いざ表示した時に文字化けするのを防いでいます。
titleに記述したものは、各ブラウザにおいてタブに表示されるものになります（このサイトだと「HTML講習のページ」）

タグ

htmlは全てタグにより構成されています。タグは必ず<タグ名>で始まり、</タグ名>とスラッシュを付けたもので終わります。このタグにはある程度特徴があり、どんな表示をさせたいかによって文章を囲むタグを変えるのです。
必須要素としては、一番大元のhtmlタグ、何かしらの前処理を行ったり、各種設定を書き込むheadタグ、そして実際に表示される内容を書き込むbodyタグです。

タグにはいくつか種類があるものの、そこまで多くはありません。使いそうなものだけ以下に列挙します。

  • <h1> <h2> <h3> <h4> <h5> <h6>
    見出しです。これで囲むだけで、他の文字よりも大きくなります。h1が一番大きく、h6まであります。このサイトでは、h2をよく使っています。

    <h1>大きいでしょ</h1>
<h2>まあね</h2>
<h3>普通くらい</h3>
<h4>小さいでしょ</h4>
<h5>小さいよね</h5>
<h6>小さすぎませんか</h6>

    大きいでしょ

    まあね

    普通くらい

    小さいでしょ

    小さいよね
    小さすぎませんか

  • <p>
    段落を表します。pタグ間では改行が行われ、少し多めに行間が開きます。

    <p>１段落目</p>
<p>２段落目</p>
<p>ほげ</p>

    １段落目

    ２段落目

    ほげ

  • <pre>
    pタグとは違って、改行やスペースの情報がそのまま表示にも反映されます。
    例えば、以下のような文字列を各タグで囲ったとき、実際の表示は右のように変化します。 

    <p>これは       pタグ
    です。</p>

<pre>これは       preタグ
    です。</pre>

    これは pタグ です。

    これは       preタグ
    です。

  • <br>
    改行を表します。pタグなどで文章を書いているとき、<br>を入れることでそこで改行されます。 

    <p>文章を書いているとき、<br>改行します。</p>

    ulで囲んだ項目は単に箇条書きにされ、olで囲んだ項目には1から順に番号がつきます。（これは、UnorderList(非順番づけ)かOrderList(順番づけ)かの違いです。）
    もちろん、liタグの中にさらにpやpreタグなどを入れることも可能です。

  • <a>
    href="URL"とすることで囲んだ文字をハイパーリンクにします。つまり、文字をクリックすると他のURLに飛ばせます。これはhrefにhtmlファイルのパスを指定すれば、ページ切り替えができるのでよく使います。

    <a href="https://google.com">Googleへはこちら</a>
    Googleへはこちら

  • <img>
    scr="ファイルへのパス"とすることで、画像を表示できます。ここでパスとは、これを記述したhtmlファイルから、目的の画像への相対パスです。また、これに関しては</img>が要りません。また、（後述がありますが）widthやheightといった属性を指定することで、大きさを変えられます。

    <img src="./images/image1.png" width=100 height=100>

タグの属性

ここまで代表的なタグを見てきましたが、これらのタグにはそれぞれ属性というものがあります。aタグのhref、imgタグのsrc,width,heightも属性です。どんな属性があるかはタグごとに決まっていて、属性はタグの機能を拡張できるものと考えるのが良いと思います。

属性は全てスペース区切りで書きます。以下、aタグを具体例に、属性によるある程度の制御を体験します。

aタグの持つ属性の一部には、href, target, donwloadがあります。

  • href・・飛ばしたい先の情報、つまりURL(https://~)や別のhtmlファイルへのパスを指定します。
  • target・・属性を追加してtarget="_blank"を指定すれば、新しいタブで開くようになります。
  • download・・この属性を追加するだけで、ページに遷移するのではなく、そのファイルをユーザーにダウンロードさせることができます。

これらを組み合わせて、機能がどう変わるかを見てみましょう。

<p><a href="./test/test.html">普通に画面が変わります。</a></p>
<p><a href="./test/test.html" target="_blank">別タブとして開きます。</a></p>
<p><a href="./test/test.html" download>押すとhtmlファイルがダウンロードできます。</a></p>

普通に画面が変わります。

別タブとして開きます。

CSSの導入

htmlだけでもある程度の機能が実現できます。でも「機能性」は増えても、「デザイン性」はあまり得られません。どうせ作るからには見やすいページにしたいということで、登場するのがCSSです。CSSを使えば、色をつけたり、文字の大きさを変えたり、表示する位置を自在に変えたり、他にも様々なことができます。

CSSは以下の3通りでhtmlと紐つけることができます。

  1. headタグ内にstyleタグを使って書く

    <head>
    <style>
        CSSの処理
    </style>
</head>

  2. 各タグのstyle属性として書く

    <p style="CSSの処理"></p>

  3. あらかじめ.cssの拡張子のファイルに記述しておいて、html側のheadタグで読み込む。（以下はdesign.cssという名前で作った時）

    <head>
    <link rel="stylesheet" type="text/css" href="design.css">
</head>

「特定の要素」の指定方法

ある文章を書いていて、「この行だけ大事だから赤くしたい」という気持ちになったとします。この時CSSで色を指定するのですが、「この行」の指定はどのように行うのでしょうか。

この時役に立つのが、属性としてのclassとid、そしてタグとしてのdivとspanです。

classとidは、あらゆるタグにつけることができる属性で、class="名前" や id="名前" のようにそのタグに名前をつけることができます。ここでの名前は自由で、自分で分かりやすい名前をつけることができます。複数の名前をつける場合、class="名前1 名前2"のように空白で区切ります。 

<p class="highlight">目立たせたい行なんです</p>
<p id="content">大事な内容を書く</p>

さらに重要なこととして、classは同じ名前を何度でも使えるのに対し、idは１度しか同じ名前を使えないという違いがあります。 

<p class="content"></p>
<p class="content"></p> 2回目でもok
<p id="title"></p>
<p id="title"></p>2回以上同じ名前を使えないのでダメ

次にdivとspanです。これはタグ単位で名前をつけるidとclassとは少し違って、より大きな、もしくは小さな範囲に適用したいときに使用します。
先程までの例では、pタグにclassやidで名前をつければ、そのタグ全体に適用されました。しかし、複数にわたるpタグ全てに同じ処理をしたいときに、全部のpタグに同じclass=""を書くのは面倒です。そこで、divで括ることにより見やすくなります。 

<p class="content">hoge1</p>
<p class="content">hoge2</p>
<p class="content">hoge3</p>
は冗長なので
<div class="content">
    <p>hoge1</p>
    <p>hoge2</p>
    <p>hoge3</p>
</div>

次にspanは、ある文章の重要キーワードだけ赤くしたいという時などに使います。 

<p>今勉強しているのは<span class="highlight">html</span>です。</p>

このように、divはより大きな範囲に、spanはより小さな範囲で名前をつけるときに役立つということです。あくまでも、名前をつける処理はidやclassにより行います。

いざ、CSSでデザインをする

どこにデザインをするかということは、上記までの方法で名前をつけることで指定できることがわかりました。ここまではhtml側での話をしていたので、いよいよCSSを書いていきます。

html側でつけた名前に対してCSSで処理をする時、CSSセレクタを使って行います。CSSセレクタは以下の規則を持っています。

  • タグ名に対しての処理は 「タグ名{ }」で行う。
  • classで付けた名前に対しては、「.class名{ }」と、ドットを付けて行う。
  • idで付けた名前に対しては、「#id名{ }」と、シャープを付けて行う。
  • あるタグや名前の中で囲まれているタグや名前に対しては、その入れ子の順（階層が浅い順）に空白を開けて記述する。(ulの中のliの中のtagクラスなら、ul li .tagみたいな)
  • あるclass名がついたタグを表すには、「タグ名.class名」とする。（空白は開けない、<p class="tag">ならp.tagみたいな)
  • 複数の要素を指定するなら、カンマで区切る。(pタグとaタグに付けたかったら、p,aみたいな)

以下、具体例です。

<!-- html -->
<pre>わーい緑</pre>
<p class="red">赤くします</p>
<p id="blue">青いですね</p>
<div class="report">
    <p  class="title">どんぐり、穴に落ちる</p>
    <p>持ち主だったおじいさんは、「悲しい」とコメントしています。</p>
    <p>もし見つけた方は、<span class="red">html警察</span>までお願いします。</p>
</div>
<!-- CSS -->
pre{              /*単純にタグに対して*/
    color:green;
}
.red{         /*class名に対して*/
    color:red;
}
#blue{     /*id名に対して*/
    color:blue;
}
.report p.title{     /*reportクラス内の、titleのクラス名を持つpタグに対して*/
    color:red;
    font-size:30px;
}

CSSセレクタを書く上では空白の扱いに気をつけなければなりません。
例えば、p .c1 ではpタグの中にあるc1クラスの要素を表しますが、p.c1ではc1のクラス名がついたpタグを表します。

p .c1だと
<p>
    < class="c1"></>
</p>
p.c1だと
<p class="c1"></p>

エスケープ文字

ここまでコード例と共に紹介してきましたが、これらのコードを画面に表示させるのには一工夫要ります。例えば、
「pタグは、<p>で始まり</p>で終わります。」
という文章を書く時、これをそのままhtmlファイルに書き込むと文章中の「<p>」や「</p>」はhtmlタグとして認識されて、「で始まり」の部分がpタグで囲まれた段落であると認識されてしまいます。

これを防ぐためにエスケープ文字を使います。代表的なエスケープ文字を以下に示します。

< > " ' &
&lt; &gt; &quot; &#x27; &amp;

よって、例えば 「<p class="hoge">文章です</p>」 と書くときには、
「&lt;p class=&quot;hoge&quot;&gt;文章です&lt;/p&gt;」 と書くことになります。この変換は手作業でやると日が暮れるので、適当にC++とかpythonとかで変換プログラムを組むと楽です。

蛇足ですが、&lt;を画面に表示するためには、これをそのままhtmlに書くと < になってしまうので、先頭の&をエスケープして&amp;lt;と書いています。ちなみにこの&amp;lt;は同様の理由で先頭の&をエスケープしないと表示されないので、&amp;amp;lt;になります・・。
